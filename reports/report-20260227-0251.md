## What Was Done

### 1. Fixed Critical Bug in `simulation_study.py` (v1)

Identified root cause of AUROC=0.347 (below chance) in the existing simulation:
- **Bug**: `compute_auroc()` sorted scores DESCENDING but used the ascending-rank Wilcoxon formula, inverting the result
- **True value**: 1 - 0.347 = 0.653 (above chance, but not reported correctly)
- **Secondary bug**: `simulate_bpfc_data()` had a term `p_agree * prob_correct + (1-p_agree) * prob_correct` that simplified to `prob_correct`, making the agreement parameter a no-op

### 2. Wrote `experiments/simulation_study_v2.py`

Corrected simulation with:
- Fixed `compute_auroc` (ascending sort, standard Wilcoxon)
- Clean independent-Bernoulli sampling model (K passes, each samples answer independently)
